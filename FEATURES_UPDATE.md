# 📦 数据持久化与下载功能说明

## ✅ 已更新的功能

### 1. 数据持久化存储

**之前的问题**: 
- 每次刷新页面或重启服务,所有模特数据都会丢失

**现在的解决方案**:
- 使用文件系统持久化存储所有数据
- 数据保存在 `backend/data/models.json`
- 服务重启后自动加载历史数据
- 所有操作(创建、生成、删除)都会实时保存

**数据存储位置**:
```
model-generator/
└── backend/
    └── data/
        └── models.json  # 所有模特数据
```

**数据结构**:
```json
{
  "model-id-1": {
    "id": "model-id-1",
    "name": "模特名称",
    "referenceImage": "/uploads/xxx.jpg",
    "createdAt": "2026-05-09T...",
    "generatedImages": [
      {
        "id": "image-id-1",
        "url": "https://...",
        "sceneImage": "/uploads/yyy.jpg",
        "prompt": "提示词",
        "createdAt": "2026-05-09T..."
      }
    ]
  }
}
```

### 2. 图片下载功能

**新增功能**:
- 每张生成的图片下方都有"💾 下载图片"按钮
- 点击即可下载图片到本地
- 文件名格式: `model-{模特名称}-{图片ID}.png`
- 自动处理跨域和下载问题

**使用方法**:
1. 生成图片后,在"已生成的图片"区域找到目标图片
2. 点击图片卡片下方的"💾 下载图片"按钮
3. 图片将自动下载到浏览器默认的下载文件夹

## 🎯 使用流程

### 创建模特 (数据会自动保存)
```
1. 点击"创建新模特"
2. 填写名称并上传图片
3. 点击"创建"
   ↓
✓ 数据已保存到 backend/data/models.json
✓ 刷新页面后数据仍然存在
```

### 生成图片 (数据会自动保存)
```
1. 选择模特
2. 上传场景图
3. 点击"生成模特图"
   ↓
✓ 生成的图片信息已保存
✓ 刷新页面后图片记录仍然存在
```

### 下载图片
```
1. 在"已生成的图片"区域找到目标图片
2. 点击"💾 下载图片"按钮
   ↓
✓ 图片下载到本地
✓ 文件名: model-模特名称-图片ID.png
```

## ⚠️ 注意事项

### 数据备份
- 定期备份 `backend/data/models.json` 文件
- 备份 `backend/uploads/` 目录中的所有图片
- 可以将整个 `backend/data` 和 `backend/uploads` 目录打包备份

### 数据安全
- `.gitignore` 已配置忽略 `data/` 目录
- 敏感数据不会提交到 Git 仓库
- 建议使用版本控制工具管理重要数据

### 性能优化
- 当前版本将所有数据保存在单个 JSON 文件
- 如果数据量很大(数百个模特),建议升级到数据库
- 图片文件单独存储在 `uploads/` 目录

## 🔧 故障排查

### Q: 数据没有保存?
A: 检查:
1. `backend/data/` 目录是否存在
2. 是否有写入权限
3. 查看控制台是否有错误信息

### Q: 刷新页面后数据丢失?
A: 检查:
1. 后端服务是否正常运行
2. `backend/data/models.json` 是否存在
3. 浏览器控制台是否有网络错误

### Q: 图片下载失败?
A: 检查:
1. 图片 URL 是否可访问
2. 浏览器是否阻止下载
3. 查看浏览器控制台的错误信息

### Q: 如何迁移数据?
A: 
1. 复制 `backend/data/models.json` 到新位置
2. 复制 `backend/uploads/` 目录中的所有图片
3. 保持相对路径结构不变
